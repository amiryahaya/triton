Recommended UX principles for unification

1
One chrome, three apps — not one giant app

Each portal stays a separate deployable (they run on different hosts and serve different audiences). They share the shell: same sidebar grammar, same topbar, same auth prompt, same footer wave, same typography, same empty/loading/error states.

2
Portal colour-coding — muted but consistent

Keep the navy/cyan base identical. Use a single 1-pixel accent per portal (purple = License, cyan = Report, lime = Manage) on the logo rail and active nav item. Users know at a glance which portal they're in without a loud theme change.

3
Role-appropriate density

License operator: ops-console density (dense tables, audit-first). Security officer: exec-summary density (big numbers, low clicks to compliance verdict). Network engineer: bulk-action density (multi-select, inline edit, CSV in/out). Same components, different defaults.

4
Cross-portal wayfinding

Topbar app-switcher shows the three portals; disabled ones are visible but greyed (e.g. Manage Server absent). A single "Triton" brand in the logo rail links back to the portal's own home — never cross-navigates silently.

5
One shared Vue component library

Ship @triton/ui as a private workspace package: Button, StatCard, DataTable, Sidebar, Topbar, EmptyState, Toast, Modal, FormField, Chart wrappers. Both portals import the same components; custom work only where a portal really needs it.
