Tradeoffs

Theme Strengths Weaknesses Best for
A · Pitch Dramatic, "command centre" feel, dense data reads well Eye fatigue on long sessions, poor for print/export, can look "gamer" 24/7 SOC wall displays, always-on dashboards
B · Slate Professional dark without the heaviness, keeps contrast for pills & charts, matches Stripe/Linear/Vercel dashboards Still dark — not ideal for executive or print Daily operator use, Manage + License portals
C · Light Professional, enterprise-standard, ready for NACSA PDF export, accessible, familiar to officials Loses the "monitoring tool" vibe, high-contrast pills need careful colour tuning Report portal for security officers, printable compliance reports

My recommendation · dual-theme system with user toggle

Build B (slate dark) as the default and C (light) as a first-class alternative. Single token system, two values per token, html[data-theme] switch. Drop pitch-dark A — it doesn't serve daily users well.

  • Theme toggle in topbar (sun/moon/system). Persists per-user, synced to localStorage & optional user profile setting.
  • Per-portal default if you want: Report → Light (officer/exec audience), License & Manage → Slate dark (operator/engineer audience).
  • Respect prefers-color-scheme on first visit before any override.
  • Exported PDFs/HTML reports always render in light theme — that's always what officials expect.
