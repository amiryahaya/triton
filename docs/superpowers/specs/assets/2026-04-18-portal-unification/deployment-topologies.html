Where should the Manage Server live?

The answer falls out of the trust boundary. Two dimensions to decide: location (on-prem vs cloud) and multi-tenancy (one customer vs many).

Architectural rule of thumb

Inside customer trust boundary

Things that initiate network actions — discover subnets, SSH into hosts, push agents. They hold credentials & need L2/L3 reach to targets.

Can live outside (cloud ok)

Things that receive data over TLS — Report Server ingests scan results pushed by agents. License Server is always vendor-operated.

One Manage Server for the whole customer. Zones are a grouping inside Manage, not separate deployments. Each zone gets its own scan profile (cpu/mem/allowlist). Manage needs L3 reach to every zone — typically sits in a management VLAN with firewall rules.

When trust domains cannot share a management channel (classified, OT, air-gapped, cross-border), each gets its own Manage Server. Same customer tenant in License Server; two Manage deployments. Report Server may also split if data residency forbids cross-zone transfer.

④ MSP / reseller — not recommended to share Manage

Manage holds SSH keys, host inventories, scan profiles — cross-tenant isolation inside one process is fragile. Instead: one Manage per customer, Report Server stays multi-tenant (it just receives signed results). MSPs deploy N Manages and one cloud Report Server.

Recommended rules — this drives the UX

  1. Manage Server is single-tenant, on-prem, per customer. It never multi-tenants across customers. Its UI does not need a tenant-switcher.
  2. Manage supports multiple network zones for one customer — zones are a first-class concept in its UI (list view, filter, per-zone scan profile). Hosts belong to exactly one zone.
  3. Report Server is multi-tenant — runs in cloud or on-prem, ingests signed results from agents across all customers. Its UI has tenant scoping (tenant admin sees their tenant only; super-admin sees all).
  4. License Server is vendor-only, always off-site from the customer. Its UI never shows customer scan data — only entitlements, seats, activations.
  5. Agents talk to Report Server directly with the licence token. Manage Server is a deployment + config plane, not a data plane. If Manage goes down, scans already scheduled still flow to Report.
  6. Air-gapped sites get their own Manage — same customer tenant in licensing, but the instances don't federate. Results either ship out via signed export or a dedicated Report Server lives inside the air-gap.
